命名规范

目录名和文件名：

文件名和目录名只能包含[a-z\d-],并以英文字母开头。

变量命名

1. 选取合适的寓意化单词，禁止 var a,b,c 这种看不懂的变量
2. 采用驼峰命名法
3. 常量全大写USER_CONFIG
4. 类名驼峰首字母大写DialogBox
5. 自定义事件名采用驼峰， onComplte

data-api


1. 全部小写用-链接
2. 全部以data开头
3. 用于展示渲染页面的容器使用data-view-detail
4. 组件标识api, data-widget-focus
5. 获取dom元素 data-ele-login-button
6. 获取和事件相关的dom元素 data-click-login-button

编码风格

一行保持在80-120个字符。

保持一个tab四个空格

花括号:

建议不要使用.net的写法

if()
{

}

保持
if() {

}
的风格

空格保持代码的清晰

good
var a = b + c;


bad
var a=b+c;

当a,b,c变量名比较长时，可能一眼望去看不清



不需要使用alignMent等插件实现对齐, 那样会浪费时间
自然空一格

good
var a = b;
var example = c;


bad
var a =       b;
var example = c;

逗号

good
{
    name: 'haha',
    age: 13,
    year: 2014
}

bad
{
    a: 'a'
   ,b: 'b'
   ,c: 'c'
}

单var 和 多var不做约束

=号

建议使用 === 不使用 == 依赖隐式类型转换
特别是处理后台返回的数据时。

null

不要用任何变量去和null比较

if


if(judge()) {

}

建议if内不要有(),所有判断条件最好用变量存一下

if 括号内的逻辑判断不要太长。

如果太长需要对逻辑判断进行拆分，尽量保证逻辑清晰


good
var passed = judge();

if(passed) {

}

建议if 不要简写

bad
if(result) alert('a'); 

good
if(result) {

}

代码中不要出现debugger

结尾统一加上;避免合并时出现问题 var a = b;

构造函数首字母大写 LoginBox

上述规范会开启jshint进行语法检测把文档生成到doc里面
